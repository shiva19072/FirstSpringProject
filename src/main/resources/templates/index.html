CRUD Operations

In a Spring Boot project, CRUD operations are the foundational actions you perform on a database. These operations—Create, Read, Update, and Delete—are essential in managing persistent data, and Spring Boot offers powerful tools to handle these efficiently using Spring Data JPA and Hibernate.

  • Create: Adds new records to the database using methods like save().
  • Read: Retrieves records from the database using methods like findAll() or findById().
  • Update: Modifies existing records in the database using the save() method with updated data.
  • Delete: Removes records from the database using deleteById().

Create

The POST operation allows users to add new student records to the database. Using an HTTP request, the server takes student data (name, college, roll number, qualification, year, and certificate) and stores it in the database, automatically generating and assigning a unique id for each student.

Retrieve

Retrieve By ID:

A GET operation retrieves a specific student's data by their unique ID. The server returns the details of the student corresponding to the provided ID, such as name, college, roll number, and other relevant details.

Retrieve All:

Another GET operation fetches all student records from the database. It provides a list of all students, showing their details in a formatted structure, useful for displaying them on the webpage or in other applications.

Update:

The PUT operation allows for the modification of an existing student's data. By providing the student's ID and updated data (e.g., change of college or year), the system overwrites the corresponding record in the database with the new values.

Delete:

The DELETE operation removes a student's record from the database. By providing the student's ID, the system identifies and deletes the specific record, ensuring the database remains up to date.
